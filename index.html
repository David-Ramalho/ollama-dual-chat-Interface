How RAG injection works:
When USE RAG is on, before each message Echo searches the loaded chunks for the closest matches to the user's last message (cosine similarity for embedding, keyword overlap for keyword mode).
The top-K results are assembled into the template — {"{context}"} is replaced with [filename] chunk text for each result.
This block is appended to the system prompt, so the model sees it before its first reply.

Good template examples:
### Relevant Notes:
{"{context}"} — neutral, good for factual files
